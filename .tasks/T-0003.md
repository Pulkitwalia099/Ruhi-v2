# T-0003 — Refactor to AI SDK v6 ToolLoopAgent

Replace the raw `streamText()` + tool call loop in `/api/chat` with the AI SDK's `ToolLoopAgent` class. This gives us a proper agent abstraction with reusable configuration, subagent support, and better loop control.

## Context

The current chat route does a single `streamText()` call with `stopWhen: stepCountIs(5)` and 5 inline tools. This works but everything is wired up ad-hoc in the route handler — model config, tools, system prompt, stop conditions, and streaming are all tangled together. The AI SDK v6 `ToolLoopAgent` class encapsulates all of this into a reusable, composable abstraction.

## Current state

`src/app/(chat)/api/chat/route.ts` — ~370 lines. The core generation logic:

```ts
const result = streamText({
  model: getLanguageModel(chatModel),
  system: systemPrompt({ requestHints, supportsTools }),
  messages: modelMessages,
  stopWhen: stepCountIs(5),
  tools: { getWeather, createDocument, editDocument, updateDocument, requestSuggestions },
  // ...
});
dataStream.merge(result.toUIMessageStream({ sendReasoning: isReasoningModel }));
```

Tools are constructed inline with closures over `session`, `dataStream`, and `modelId`.

## What changes

### 1. Define a chat agent

Create `src/lib/ai/agent.ts` with a `ToolLoopAgent`:

```ts
import { ToolLoopAgent } from "ai";

export function createChatAgent({ model, tools, instructions, ...opts }) {
  return new ToolLoopAgent({
    model,
    instructions,
    tools,
    stopWhen: stepCountIs(10),
    ...opts,
  });
}
```

The agent encapsulates model, instructions, tools, and loop control in one place. Can be instantiated per-request with the right model/tools for that user's session.

### 2. Use `createAgentUIStreamResponse` in the route

Replace the manual `createUIMessageStream` + `streamText` + `dataStream.merge` pattern with:

```ts
return createAgentUIStreamResponse({
  agent: chatAgent,
  uiMessages,
});
```

This handles streaming, message conversion, and the agent loop automatically.

### 3. Move tool construction out of the route

Tools currently close over `session`, `dataStream`, `modelId` — they're constructed inline in the route. Refactor so tools are defined as standalone factories that the agent can use. The `dataStream` dependency is the tricky part — `createAgentUIStreamResponse` manages the stream internally, so tools that write to the data stream (artifact deltas, suggestions) need to use the `execute` generator pattern (`async function*` with `yield`) instead.

### 4. Subagent opportunities

Once the primary agent works, we can split off specialized subagents:

- **Document agent** — handles `createDocument`, `editDocument`, `updateDocument` with its own instructions about artifact handling. Could use a different model optimized for code/writing.
- **Research agent** — web search, data retrieval (future). Runs in its own context window to avoid bloating the main conversation.
- **Suggestion agent** — `requestSuggestions` currently uses structured output via a separate `streamObject` call inside the tool. Could be a subagent with its own model.

Each subagent gets its own context window, so expensive operations (like generating a full document) don't eat into the main conversation's context.

### 5. Loop control improvements

Current: `stepCountIs(5)` — hard limit of 5 steps.

With `ToolLoopAgent` we can use `prepareStep` for:
- **Dynamic model selection** — use a cheaper model for simple tool calls, stronger model for final synthesis
- **Context management** — trim message history if it grows too large
- **Tool phasing** — enable/disable tools based on step number (e.g., only allow `createDocument` after initial conversation)

## Key API surfaces

| API | Purpose |
|---|---|
| `new ToolLoopAgent({ model, instructions, tools, stopWhen })` | Define a reusable agent |
| `agent.generate({ prompt })` | One-shot generation |
| `agent.stream({ prompt })` | Streaming generation |
| `createAgentUIStreamResponse({ agent, uiMessages })` | Stream agent response to UI (replaces manual `createUIMessageStream` + `streamText`) |
| `stopWhen: stepCountIs(n)` | Max step limit |
| `prepareStep: ({ stepNumber, messages }) => {}` | Per-step config (model switching, context trimming, tool selection) |
| `async function* execute()` + `yield` | Streaming subagent progress via preliminary tool results |
| `toModelOutput` on tools | Control what the parent agent sees from subagent results (context compression) |
